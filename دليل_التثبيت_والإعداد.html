🗄️ الجزء الأول: تثبيت وإعداد SQL Server Express

  1. تحميل SQL Server Express افتح المتصفح وانتقل إلى:
    https://www.microsoft.com/sql-server/sql-server-downloads
    اختر Express (مجاني) ثم قم بتشغيل المثبّت.
  2. خيارات التثبيت عند فتح المثبّت اختر "Basic" للتثبيت السريع. سيتم تثبيت SQL Server تلقائياً باسم instance: SQLEXPRESS
  3. التأكد من تشغيل الخدمة
    اضغط Win + R ثم اكتب services.msc ثم Enter.
    ابحث عن SQL Server (SQLEXPRESS) وتأكد أن الحالة Running.
    إذا كانت متوقفة، انقر عليها بزر الفأرة الأيمن ← Start.
  4. تفعيل الاتصال عبر Named Pipes (اختياري) افتح SQL Server Configuration Manager ← SQL Server Network Configuration ← Protocols for SQLEXPRESS ← تفعيل Named Pipes و TCP/IP.
✅ ملاحظة: البرنامج يُنشئ قاعدة البيانات POSSystem تلقائياً عند أول تشغيل — لا تحتاج لإنشائها يدوياً.

⚙️ الجزء الثاني: إعداد الاتصال بقاعدة البيانات

يوجد بجانب البرنامج ملف ConnectionString.txt يحتوي على بيانات الاتصال بـ SQL Server.

الإعداد الافتراضي (SQL Server Express):

Server=.\SQLEXPRESS;Trusted_Connection=True;TrustServerCertificate=True;

بدائل حسب الإعداد:

الحالةسلسلة الاتصال
SQL Server Express (افتراضي) Server=.\SQLEXPRESS;Trusted_Connection=True;TrustServerCertificate=True;
SQL Server Default Instance Server=.;Trusted_Connection=True;TrustServerCertificate=True;
مع اسم مستخدم وكلمة مرور Server=.\SQLEXPRESS;User Id=sa;Password=XXXX;TrustServerCertificate=True;
SQL Server على جهاز آخر في الشبكة Server=192.168.1.100\SQLEXPRESS;Trusted_Connection=True;TrustServerCertificate=True;
⚠️ تنبيه: إذا عدّلت الملف، تأكد من حفظه بترميز UTF-8 واحذف أي مسافات زائدة في النهاية.

🚀 الجزء الثالث: تشغيل التطبيق

الطريقة الأولى: عبر Visual Studio (للمطورين)

  1. فتح المشروع انتقل إلى مجلد البرنامج وافتح الملف:
    POSSystem.sln
  2. استعادة الحزم في Visual Studio من القائمة: Tools → NuGet Package Manager → Restore All Packages
    أو عبر Command Prompt:
    dotnet restore
  3. بناء المشروع
    :: من القائمة: Build → Build Solution (أو Ctrl+Shift+B) :: أو من Command Prompt: dotnet build
  4. تشغيل البرنامج
    :: من Visual Studio: F5 (أو زر Run الأخضر) :: أو من Command Prompt: dotnet run --project POSSystem\POSSystem.csproj

الطريقة الثانية: نشر EXE مستقل (للاستخدام النهائي)

لإنشاء ملف تنفيذي يعمل بدون Visual Studio:

:: افتح Command Prompt في مجلد المشروع وشغّل: dotnet publish POSSystem\POSSystem.csproj ^ -c Release ^ -r win-x64 ^ --self-contained true ^ -o output\POSSystem

ستجد ملف POSSystem.exe في مجلد output\POSSystem — يمكن نسخه إلى أي جهاز بدون تثبيت .NET.

💡 ملاحظة: تأكد من وجود ملف ConnectionString.txt في نفس مجلد EXE عند النشر.

🔧 الجزء السادس: حل المشكلات الشائعة

❌ "A network-related or instance-specific error"

السبب: خدمة SQL Server متوقفة أو اسم الـ instance خاطئ.
الحل:
  1. افتح Services.msc وتأكد أن SQL Server (SQLEXPRESS) يعمل.
  2. عدّل ملف ConnectionString.txt وجرّب: Server=localhost\SQLEXPRESS

❌ "Login failed for user"

السبب: Windows Authentication غير مفعّل أو الحساب ليس له صلاحية.
الحل: افتح SQL Server Management Studio (SSMS) ← Security ← Logins ← تأكد من وجود حسابك بصلاحيات sysadmin.

❌ خطأ في البناء Build Failed

الحل:
dotnet restore dotnet build
إذا استمر الخطأ، احذف مجلد bin و obj ثم أعد البناء.

❌ رسالة "الترخيص منتهي"

الحل: افتح SSMS وشغّل:
UPDATE POSSystem.dbo.AppSettings SET LicenseStatus = 'active';
